## 为什么是 Nestjs

~~因为我学了 Nestjs 之后，就能写 SpringBoot 的项目。😂😂~~

> 因为 Nestjs 的设计思路和 SpringBoot 非常相似。

如果你在工作当中，经常接触到的后端是 Java 的 SpringBoot，那么选择 Nestjs 作为你的第一个后端框架就不会错。

Nestjs 使用 Typescript 作为开发语言，当你后续接触其他强类型语言也会更容易上手。

为什么与 SpringBoot 相似是必要的，因为大部分公司的后端用的都是 SpringBoot 啊。😂😂

| 对比项   | Nestjs           | SpringBoot |
| -------- | ---------------- | ---------- |
| 语言     | Typescript       | Java       |
| 框架     | Express/ Fastify | Spring     |
| 模块     | 模块             | 包         |
| 控制器   | Controller       | Controller |
| 服务     | Service          | Service    |
| 依赖注入 | 依赖注入         | 依赖注入   |
| 中间件   | 中间件           | 中间件     |

学会 Nestjs 之后，你就能够看懂 SpringBoot 的项目了，甚至可以去写。

## 更重要地文档

> 你可以在 Nestjs 的官方文档中找到后端开发的所有知识。

Nestjs 的文档对于前端开发人员非常友好，他的排版风格和各种前端框架的文档一样，像 Vue、React 等等。

当然也少不了各位大佬翻译的中文文档。

通常认为学习后端开发还需要了解数据库、缓存、消息队列、安全、websocket、微服务等等。

~~没错，我就是顺着 Nestjs 的文档去恶补的这些内容~~

但是在 Nestjs 中，这些都在 Nestjs 的文档里，你只需要学习 Nestjs 就可以了。

你可以在 Nestjs 的官方文档中找到这些知识的简单介绍。

如果你想深入了解这些知识，那么你需要去专门学习。

但如果只是使用，那么 Nestjs 已经帮你解决了这些问题。

~~其他的用的时候再说呗~~

也许你会问是否需要学 Express、Fastify 或者 Koa，不，你不需要。

你只需要 Nestjs 。

## 以及非常重要的脚手架

> 没有脚手架的框架是不完整的。

Nestjs 官方提供了非常好的脚手架，你可以使用它来快速创建项目。

你不仅可以用这个脚手架来创建项目，你还可以用它来生成任何项目里需要的东西，是的任何。

当然也包括 CRUD 最爱的 CRUD generator。

其实 Nestjs 还提供了一个 DEVTOOLS。

它可以帮你做很多事情。模块关系图、类关系图，还有沙盒、性能分析器。
